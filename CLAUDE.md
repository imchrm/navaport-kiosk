# CLAUDE.md — инструкции для Claude Code

Этот файл задаёт правила работы в репозитории. Соблюдать при любых изменениях.

## Сначала прочитать внутри папки `docs/` следующие файлы
1. `CONTEXT.md` — снимок проекта и среда.
2. `ARCHITECTURE.md` — структура и модель контента в типах.
3. `TODO.md` — активная фаза и открытые решения.
4. `CHANGELOG.md` — 

## Жёсткие правила
- Файлы создавать в кодировке **UTF-8**, перевод строки **LF**.
- **Эмодзи** в коде, скриптах и контенте не использовать.
- Среда разработки — Windows; команды и примеры писать под **PowerShell**.
- TypeScript всегда `strict`. Не вводить `any`; для неизвестного входа — `unknown` + сужение.

## Конвенции кода
- Варианты и состояния — **дискриминированные union** по полю `kind`, не наборы опциональных флагов.
- Узел меню — `branch` xor `leaf` на уровне типа; не «node с опциональными `children`/`target`».
- Все `switch`/обработки по `kind` закрывать `assertNever` (exhaustiveness). Новый вариант должен ломать компиляцию, пока не обработан.
- Данные извне процесса (контент-JSON, IPC) валидировать **Zod на границе**, один раз; дальше типы доверенные.
- IPC main <-> renderer — только через типизированный фасад из `contract`, без строковых каналов в коде.
- Контентные структуры — `readonly` по умолчанию.

## Процесс
- По **открытым решениям D1-D5** (`ARCHITECTURE.md`) **не угадывать** — поднимать вопрос. Если решение блокирует задачу, сказать об этом, а не выбирать молча.
- Держать документы в синхроне: завершённый пункт `TODO.md` → пометить `[x]` и перенести в `CHANGELOG.md`.
- Уважать разделение rolling/stable: «как навсегда» — в `ARCHITECTURE.md`; «где сейчас» — в `CONTEXT.md`/`TODO.md`. Не дублировать.
- Следовать порядку фаз из `ARCHITECTURE.md` §12; начинать с активной фазы в `TODO.md`.
