# Airport Wayfinding Kiosk

Информационный киоск аэропорта на сенсорном экране (landscape, режим 24/7, полностью офлайн). Три способа найти нужное: attract-loop видео на простое, иерархичное меню «по цели» и 2D-план этажа «по месту». Оба входа ведут в 360-туры. Языки интерфейса: `uz`, `ru`, `en`.

> **Статус:** pre-MVP, Фаза 0 (каркас). Репозиторий ещё разворачивается — см. `TODO.md`.

---

## Возможности

- Attract-loop видео на простаивающем экране до первого касания.
- Иерархичное меню (2-3 уровня), drill-down по тапу, без hover.
- 2D-план этажа с кликабельными зонами (входит в MVP).
- Встроенные 360-туры по локациям (panotour viewer).
- Мультиязычность `uz` / `ru` / `en`, переключение на лету.
- Контент (пункты меню, зоны) редактируется без программиста.
- Режим киоска: idle-сброс, блокировки жестов, авто-старт, работа сутками.

---

## Стек

Electron · React · Vite · TypeScript (strict) · Zod · npm workspaces. 360-движок — `panotour` viewer (Vanilla JS), встраивается как зависимость. Подробности и обоснование — в `ARCHITECTURE.md`.

---

## Структура (кратко)

```
packages/
  contract/   типы IPC main <-> renderer
  content/    модель контента: типы, zod-схемы, загрузчик, данные, i18n
  main/        Electron main process: окно, kiosk lifecycle, IPC
  renderer/   Vite + React + TS: экраны (attract, menu, map, tour-host)
vendor/        встроенный panotour viewer
assets/        видео, SVG-планы, тайлы туров
content/       nav.json, maps.json, i18n.json
```

Полная схема и границы подсистем — `ARCHITECTURE.md` §3-4.

## Установка и запуск

Скрипты устанавливаются в Фазе 0. Целевой набор:

```powershell
npm install          # установка зависимостей по всем пакетам
npm run typecheck    # проверка типов по всем пакетам
npm run dev          # запуск renderer + Electron в режиме разработки
npm run build        # сборка
npm run package      # пакетирование под целевую ОС киоска
```

До завершения Фазы 0 часть скриптов может отсутствовать — актуальный статус в `TODO.md`.
