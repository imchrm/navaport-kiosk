## Снимок проекта

Информационный киоск аэропорта на сенсорном экране (portrait, 24/7, офлайн). Три способа найти нужное: attract-loop видео на простое, иерархичное меню «по цели» (2-3 уровня) и 2D-план этажа «по месту». Оба входа ведут в 360-туры (panotour). Языки: `uz`, `ru`, `en`.

Контекст поставки: клиент/тендер. Деливерабл — рабочий киоск плюс возможность для клиента менять пункты меню и зоны без программиста.

## Среда и инструменты

- **Разработка:** Windows 11, PowerShell-native команды, VS Code. При работе с удалёнными ресурсами — Remote-SSH.
- **Рантайм проекта:** Node.js + npm workspaces, Electron, Vite, TypeScript strict.
- **Целевой рантайм (киоск): Windows x64** (D1 закрыт). electron-builder собирает NSIS-инсталлятор и portable .exe.
- **Связанный актив:** `panotour` viewer — встраивается через `<iframe>`, не переписывается. Туры в prod раздаёт локальный HTTP-сервер (127.0.0.1:51440).

Команды в документации и скриптах писать под PowerShell. Файлы создавать в UTF-8. Эмодзи в коде и скриптах не использовать.

## Конвенции кода

Подробности структуры — в `ARCHITECTURE.md`. Здесь — рабочие принципы:

- TypeScript `strict: true`; идентификаторы и комментарии в коде — на английском.
- Состояния и варианты моделировать **дискриминированными union** (`kind`), а не наборами опциональных флагов. Узел меню — `branch` xor `leaf`, не «node с опциональными children/target».
- Обработку вариантов закрывать **exhaustive** (`assertNever`) — добавление нового `kind` должно ломать компиляцию, пока его не обработают.
- Данные извне процесса (контент-JSON, IPC) валидировать **Zod на границе**, один раз. Дальше типы считаются доверенными.
- IPC main ↔ renderer — только через типизированный фасад из `contract`, без «магических строк» каналов.
- `readonly` по умолчанию для контентных структур.

## Глоссарий

- **attract-loop** — зацикленное видео на простаивающем экране до первого касания.
- **drill-down** — углубление в меню по тапу (без hover, экран сенсорный).
- **две оси навигации** — «по цели» (меню) и «по месту» (карта); обе ведут в туры.
- **NavTarget** — назначение листа/зоны: `tour` | `info` | `location` | `external`.
- **tour / scene** — тур = граф панорам; scene = конкретная панорама-точка входа.
- **zone** — кликабельная область на 2D-плане, привязанная к `NavTarget`.
- **floor map** — SVG-план этажа с зонами.
- **panotour viewer** — внешний 360-движок, ведёт навигацию внутри тура.
- **rolling / stable** — режимы документов: меняющийся снимок vs устойчивая опора.

## Принятые решения (кратко)

Полный список развилок (включая открытые D4, D5) — в `ARCHITECTURE.md` → «Открытые решения». Зафиксировано:

- Оболочка — Electron (не браузер в kiosk-режиме).
- Renderer — React + Vite + TypeScript strict; состояние — `useReducer` + дискриминированные actions.
- Монорепо — npm workspaces; пакеты `contract` / `content` / `main` / `renderer`.
- Навигация — две оси (меню + 2D-карта); **2D-карта в MVP**.
- Ориентация — portrait (1080×1920 prod, 540×960 dev); языки — `uz`, `ru`, `en`.
- D1: целевая ОС — **Windows x64**.
- D2: узбекский — **латиница**.
- D3: видео-фон всегда играет; при нажатии кнопки пауза + затемнение; idle 5 с (dev) / 60 с (prod) → attract.
- D6: 360-туры через `<iframe>`, не через vendor/ subtree.
